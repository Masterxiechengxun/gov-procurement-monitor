# 政府采购信息自动监控系统

自动全量抓取政府采购网站的所有采购公告信息，存入云数据库供微信小程序浏览。用户在小程序中设置自己感兴趣的设备关键字，系统自动匹配并推送命中的采购信息到微信。

## 功能特性

- **全量抓取**: 按策略自动全量抓取政府采购网站的所有采购公告（不按关键字过滤），确保数据完整性
- **内置双来源**: 内置中国政府采购网（CCGP）和四川省公共资源交易网两个数据来源，可在设置页独立开关启用/停用
- **智能匹配**: 抓取入库时用所有用户的合并关键字打标签，方便前端快速筛选展示
- **扁平关键字**: 用户管理设备关键字，每个关键字即独立分类，自动同步为首页筛选标签和统计卡片
- **精准推送**: 全量抓取后，按每个用户自己的关键字独立过滤，只推送该用户真正关注的采购信息
- **小程序浏览**: 在微信小程序内查看全部采购信息，支持多维度筛选
- **数据统计**: 可视化数据分析面板 — 7天趋势图、来源分布、关键词排行、采购类型分布、多分类占比
- **NEW 标记**: 今日新增信息标注醒目的 NEW 标签
- **关键词高亮**: 匹配的设备关键字红色高亮显示
- **用户隔离**: 多用户独立配置，所有个人设置均按 openid 隔离存储在云端。抓取聚合所有用户启用的来源，推送按各用户关键字独立过滤
- **跨设备同步**: 个人配置存储在云数据库，换设备登录自动同步
- **详细日志**: 按来源展示抓取成功/失败状态，错误实时通知
- **存储用量**: 云数据库使用量可视化（采购总数、关注匹配、今日新增、容量占比）
- **专业界面**: Trust & Authority 设计风格，骨架屏加载、入场动画、精美空白状态页
- **零成本运维**: 基于微信云开发免费额度，无需额外服务器

## 技术栈

- 微信小程序原生框架 (WXML + WXSS + JavaScript)
- 微信云开发 (云函数 + 云数据库)
- Node.js 18 + axios + cheerio
- PushPlus 微信推送

## 部署步骤

### 第一步: 准备工作

1. **注册微信小程序账号**
   - 访问 [微信公众平台](https://mp.weixin.qq.com)
   - 点击「立即注册」→ 选择「小程序」→ 选择「个人」类型
   - 按提示完成注册（免费）
   - 注册完成后，在「开发管理」→「开发设置」中获取 **AppID**

2. **安装微信开发者工具**
   - 访问 [下载页面](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
   - 下载对应系统版本并安装

3. **注册 PushPlus 获取 Token**
   - 访问 [PushPlus](https://www.pushplus.plus/)
   - 微信扫码登录
   - 在首页复制您的 **Token**（一串字母数字组成的字符串）

4. **安装 Node.js**
   - 访问 [Node.js 官网](https://nodejs.org/)
   - 下载 LTS 版本并安装

### 第二步: 导入项目

1. 打开微信开发者工具
2. 选择「导入项目」
3. 项目目录选择本项目的根目录 `InfoCrawler`
4. AppID 填写您在第一步获取的小程序 AppID
5. 点击「导入」

### 第三步: 配置 AppID

打开 `project.config.json`，将 `appid` 字段修改为您的小程序 AppID:

```json
{
    "appid": "wx1234567890abcdef"
}
```

### 第四步: 开通云开发

1. 在微信开发者工具中，点击工具栏的「云开发」按钮
2. 点击「开通」，选择免费的基础版
3. 创建一个云开发环境（记住环境 ID）

### 第五步: 创建数据库集合

在云开发控制台 → 数据库中，创建以下三个集合:

| 集合名称 | 用途 |
|----------|------|
| `procurements` | 存储采购信息（全局共享） |
| `crawl_log` | 存储抓取日志（系统级） |
| `config` | 存储用户配置（按 openid 隔离） |

对每个集合，点击「权限设置」，选择「所有用户可读，仅创建者可读写」。

### 第六步: 部署云函数

在微信开发者工具中，依次右键点击各云函数目录 → 选择「上传并部署：云端安装依赖」:

1. `cloudfunctions/crawl` — 列表抓取（第一阶段）
2. `cloudfunctions/fetchDetail` — 详情补抓（第二阶段）
3. `cloudfunctions/getData` — 数据查询
4. `cloudfunctions/clean` — 数据清理

> **重要**: 部署完成后，需要在云开发控制台 → 云函数中手动设置每个函数的超时时间（默认仅 3 秒会导致执行失败）:
> - `crawl`: **60 秒**（最多抓 15 页，约 45s；若需抓更多页可调至 300 秒）
> - `fetchDetail`: **60 秒**（每批最多 8 条，每条 5s 超时 + 0.8s 间隔）
> - `getData`: **60 秒**
> - `clean`: **60 秒**

### 第七步: 创建数据库索引

在云开发控制台 → 数据库 → `procurements` 集合 → 「索引」中，添加以下索引（加速查询）:

| 字段 | 方向 | 用途 |
|------|------|------|
| `url` | 升序 | 去重查询加速 |
| `contentHtml` | 升序 | fetchDetail 查询空值记录加速 |
| `crawledAt` | 升序 | fetchDetail 按入库时间排序加速 |

### 第八步: 设置定时触发器

在云开发控制台 → 云函数中:

**crawl 函数** (每小时整点触发，由策略判断是否执行):
1. 点击 `crawl` 函数 → 「触发器」
2. 添加触发器，选择「定时触发」
3. 触发周期填写 Cron 表达式: `0 0 * * * * *`
4. 点击保存
5. 注意: 函数内部会根据用户设置的抓取策略（间隔、日期类型、开始时间）自动判断是否执行

**fetchDetail 函数** (每小时第 5 分钟触发，在 crawl 之后运行):
1. 点击 `fetchDetail` 函数 → 「触发器」
2. 添加触发器，选择「定时触发」
3. 触发周期填写 Cron 表达式: `0 5 * * * * *`
4. 点击保存

**clean 函数** (每周日凌晨 3 点清理旧数据):
1. 点击 `clean` 函数 → 「触发器」
2. 添加触发器，选择「定时触发」
3. 触发周期填写 Cron 表达式: `0 0 3 * * 0 *`
4. 点击保存

### 第九步: 配置 PushPlus Token

1. 在微信开发者工具中预览小程序（或真机调试）
2. 进入「设置」页面
3. 在 PushPlus Token 输入框中粘贴您的 Token
4. 点击「保存Token」

### 第十步: 测试

1. 在设置页面点击「手动触发一次抓取」
2. 等待几分钟
3. 回到首页查看是否有数据
4. 如果有匹配关注分类的采购信息，检查微信是否收到 PushPlus 推送

### 第十一步: 提交审核发布

1. 在微信开发者工具中，点击「上传」
2. 填写版本号和备注
3. 登录 [微信公众平台](https://mp.weixin.qq.com)
4. 在「管理」→「版本管理」中提交审核
5. 审核通过后点击「发布」

## 数据架构

### 数据隔离模型

| 数据 | 集合 | 隔离方式 | 说明 |
|------|------|----------|------|
| 采购信息 | `procurements` | 全局共享 | 所有用户查看相同数据 |
| 抓取日志 | `crawl_log` | 系统级 | 无用户归属 |
| PushPlus Token | `config` | 按 openid | 每用户独立 |
| 启用来源列表 | `config` | 按 openid | 每用户独立（控制抓取哪些网站） |
| 设备关键字 | `config` | 按 openid | 每用户独立（同时决定关注分类） |
| 抓取策略 | `config` | 按 openid | 每用户独立 |
| 关注分类 | `config` | 按 openid | 每用户独立（由关键字自动同步，无需手动维护） |

### 抓取与匹配流程

```
定时/手动触发
    ↓
从数据库加载所有用户配置（网站、关键字、策略、推送Token）
    ↓
聚合所有用户的网站列表 → 确定本次要抓取哪些网站
    ↓
全量抓取每个网站的所有采购公告（不按关键字过滤）
    ↓
去重 → 合并所有用户关键字打标签(isChemical/matchedKeywords) → 全部入库
    ↓
遍历每个用户：用该用户自己的关键字过滤新增数据 → 命中的推送到该用户微信
```

- **抓取阶段**: 全量获取，不做任何关键字过滤
- **打标签阶段**: 用所有用户合并后的关键字给条目打标签，仅用于前端筛选展示
- **推送阶段**: 每个用户独立过滤，只推送匹配该用户关注关键字的条目
- **前端展示**: 用户在小程序中只看到自己关注分类匹配的采购信息

### 云端 + 本地缓存双层架构

个人配置采用「本地缓存 + 云端持久化」策略：
- 页面加载时先读本地缓存（同步、快速），保证首屏不阻塞
- 后台异步从云端拉取最新数据，更新本地缓存和页面
- 保存时同时写本地缓存和云端，确保跨设备同步

## 自定义配置

以下配置均可在小程序「设置」页面直接操作，无需修改代码：

- **采购网站**: 内置中国政府采购网和四川省公共资源交易网，每个来源独立开关控制是否抓取，至少保留一个启用
- **设备关键字**: 扁平关键字列表，支持添加/删除；每个关键字自动成为一个筛选分类，全部同步为关注分类
- **抓取策略**: 设置是否启用自动抓取、执行间隔（1/2/4/6/8/12/24 小时）、执行日期（每天/仅工作日/仅周末）；云函数触发器时间固定不可改
- **推送通知**: 配置 PushPlus Token，命中关键字的新数据自动推送到微信

### 设置页面卡片顺序

1. 采购网站 — 开关控制各来源网站的启用/停用
2. 设备关键字 — 扁平关键字管理，每个关键字即独立筛选分类
3. 自动抓取策略 — 查看云函数触发器（只读），配置执行间隔和日期
4. 手动抓取 — 手动触发抓取和查看上次抓取记录
5. 推送通知 — 配置 PushPlus Token
6. 数据管理 — 查看云数据库使用情况和统计数据

### 关键字即分类机制

- 设置页「设备关键字」卡片管理扁平关键字列表（如：耗材、试剂、仪器）
- 每个关键字自动对应一个同名分类，无需单独维护分类
- 关键字列表变更时自动同步：关注分类 = 全部关键字
- 首页筛选标签栏自动渲染（全部 | 耗材 | 试剂 | … | 今日新增）
- 首页统计卡片区和统计页独立展示每个关键字分类的数据
- 筛选页内容筛选区按每个关键字分类独立控制

### 开发新的采购网站爬虫

`sources.js` 作为爬虫技术注册表（baseUrl、请求参数等实现细节）。用户在设置页可对每个内置来源独立开关启用/停用。

如果需要接入一个目前未支持的全新采购网站（如某市级公共资源交易网），需要开发对应的爬虫代码：

1. 在 `cloudfunctions/crawl/crawlers/` 下参照现有爬虫新建文件（爬虫应全量抓取，不做关键字过滤）
2. 在 `cloudfunctions/crawl/config/sources.js` 中注册该网站的技术配置
3. 在 `cloudfunctions/crawl/index.js` 的 `CRAWLER_MAP` 中注册新爬虫类
4. 在 `miniprogram/pages/settings/settings.js` 的 `_builtInSources` 数组中添加对应条目（id、name、website、desc）
5. 重新部署 crawl 云函数，用户进入设置页即可看到新来源并开关控制

## 常见问题

**Q: 抓取失败怎么办？**
A: 政府网站可能临时维护或更新了页面结构。查看云函数日志了解具体错误，如需修复请更新对应的爬虫代码。

**Q: 为什么小程序内不能直接打开政府网站？**
A: 个人类型小程序不支持 web-view 组件，无法内嵌外部网页。您可以复制链接后在浏览器中打开。PushPlus 推送的消息中包含可直接点击的原文链接。

**Q: 数据保留多久？**
A: clean 云函数每周日凌晨 3 点自动清理旧数据，默认保留 90 天。如需调整，可在云开发控制台直接修改 `config` 集合中 `key = "retention_days"` 的记录值。

**Q: 如何修改自动抓取时间？**
A: 在小程序「设置」→「自动抓取策略」的「执行控制」区域设置启用开关、抓取间隔（1~24 小时）和执行日期（每天/工作日/周末）。云函数触发器固定为每小时整点触发，函数内部根据用户策略判断是否真正执行。

**Q: 多人使用时数据会冲突吗？**
A: 不会。采购数据全量抓取后共享存储，所有用户可浏览全部数据。每个用户的个人配置按 openid 隔离存储。抓取时聚合所有用户配置的网站全量抓取，推送时按每个用户自己的关键字独立过滤后单独发送，互不干扰。

**Q: 换设备登录设置会丢失吗？**
A: 不会。所有个人配置存储在云端数据库，按微信 openid 关联。换设备登录后会自动从云端同步所有设置。

**Q: 抓取失败了怎么知道？**
A: 当抓取过程中有网站请求失败时，系统会通过 PushPlus 向所有配置了 Token 的用户发送错误报告。设置页的「上次抓取记录」也会显示各网站的成功/失败状态。
